# 다수 소방서 확장을 위한 배포 및 DB 분리 가이드

이 문서는 김해동부소방서 외의 다른 소방서(밀양, 창녕 등)에서도 독립적으로 앱을 사용할 수 있도록 시스템을 확장하는 방법을 설명합니다. 나중에 작업을 시작하실 때 이 가이드를 참고하세요.

## 1. 개요
가장 효율적인 방식은 **'하나의 소스 코드'**를 유지하면서, 각 소방서별로 **'개별 배포 환경(Vercel/Cloudflare)'**과 **'개별 데이터베이스(Supabase)'**를 구성하는 것입니다.

## 2. 작업 순서 (Checklist)

### 2.1 코드 환경 변수화 (Refactoring)
현재 코드에 직접 적혀 있는 설정값들을 외부에서 주입받을 수 있게 수정해야 합니다.
- `src/lib/supabase.js`: `SUPABASE_URL`과 `SUPABASE_KEY`를 `import.meta.env`를 사용하도록 수정.
- `src/constants/index.js`: 지역별 `DISTRICTS`(관할구역)와 `HOSPITALS`(병원) 정보를 분리.

### 2.2 부서별 배포 (Hosting)
1. **GitHub 저장소**: 현재 코드를 GitHub에 업로드합니다.
2. **Vercel 프로젝트 생성**:
   - `fire-command-miryang` (밀양용)
   - `fire-command-changnyeong` (창녕용)
   - 각 프로젝트 생성 시 동일한 GitHub 저장소를 연결합니다.
3. **Environment Variables 설정**: 각 프로젝트의 설정에서 해당 소방서의 DB 주소를 입력합니다.

### 2.3 데이터베이스(Supabase) 복제
1. **신규 프로젝트 생성**: Supabase에서 새로운 프로젝트를 만듭니다.
2. **스키마 복사**: 기존 DB의 테이블 구조를 SQL로 추출(Export)하여 새 프로젝트의 SQL Editor에서 실행합니다.
3. **지역 데이터 업데이트**: 해당 지역의 소화전(fire_hydrants.json) 데이터를 새 환경에 맞게 업로드합니다.

## 3. 필요 정보 (미리 준비할 것)
나중에 작업을 시작할 때 다음 정보를 준비해 주시면 제가 바로 적용해 드릴 수 있습니다:
- **대상 소방서 명칭** (예: 밀양소방서)
- **지도 중심점 좌표** (위도, 경도)
- **관할 119안전센터 및 행정동 목록**
- **인근 응급의료기관(병원) 목록 및 좌표**
